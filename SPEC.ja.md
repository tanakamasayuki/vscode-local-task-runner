# Local Task Runner - 仕様書

## 概要

ツール名: `Local Task Runner`

リポジトリ: <https://github.com/tanakamasayuki/vscode-local-task-runner>

実装方針: `JavaScript only`（TypeScript 不使用、トランスパイル工程なし）

アクティブファイルのディレクトリと、その親ディレクトリ階層（ワークスペースルートまで）にある実行可能スクリプトを自動検出し、サイドバーからワンクリックで実行できる VS Code 拡張機能。

この拡張機能はストア公開を想定しつつ、個人ワークフローの効率化に最適化する。

------------------------------------------------------------------------

## スクリプト検出

### スキャン範囲

アクティブエディタが切り替わったとき、拡張機能は次をスキャンする:

-   アクティブファイルのディレクトリ
-   その直接の親ディレクトリを順に
-   ワークスペースフォルダのルートまで

各ディレクトリでは直下の内容のみをスキャンする（再帰検索なし）。

アクティブファイルがワークスペースに含まれない場合:

-   そのファイル自身のディレクトリのみをスキャンする。

表示順:

-   各ディレクトリ内のスクリプトはファイル名の昇順でソートする

------------------------------------------------------------------------

## 対応スクリプト種別（MVP）

スクリプトは OS ごとにフィルタする:

### Windows

-   `.bat`
-   `.cmd`
-   `.ps1`

### macOS / Linux

-   `.sh`

現在の OS に適した拡張子のみを表示する。

------------------------------------------------------------------------

## PowerShell 実行戦略（.ps1）

`.ps1` ファイルは次のコマンドで実行する:

powershell.exe -NoProfile -NonInteractive -ExecutionPolicy Bypass -File
"`<scriptPath>`"

理由:

-   ユーザー設定を不要にする
-   システムの実行ポリシーを変更しない
-   プロファイル起因の副作用を回避する
-   実行ブロック問題を最小化する

------------------------------------------------------------------------

## 実行モデル

スクリプトは `vscode.tasks.executeTask` を使って VS Code タスクとして実行する。

### 実行動作

-   `cwd` はアクティブファイルのディレクトリに設定する
-   共有ターミナルパネルを使用する
-   複数回実行を許可する（MVP では重複排除なし）
-   クリック時の確認ダイアログは表示しない
-   追加のログ出力チャネルは持たず、標準のタスク実行出力のみを使用する

------------------------------------------------------------------------

## サイドバー UI 挙動

### 構造

スクリプトはディレクトリ階層ごとにグループ化する:

-   `./`（アクティブファイルのディレクトリ）
-   `../`
-   `../../`
-   ...
-   `<workspace root>`

各ディレクトリはツリーグループとして表示する。デフォルトで展開するのはカレントディレクトリのみ。

------------------------------------------------------------------------

## リフレッシュ挙動

スクリプト一覧が更新されるタイミング:

-   アクティブエディタが変更されたとき

------------------------------------------------------------------------

## キーバインド

### 基本実行

-   クリック → スクリプト実行（引数なし）

### キーボードショートカット

カレントディレクトリ内のスクリプトのみ対象:

-   Ctrl+Shift+1 → 1番目のスクリプトを実行
-   Ctrl+Shift+2 → 2番目のスクリプトを実行
-   ...
-   Ctrl+Shift+9 → 9番目のスクリプトを実行

キーバインドは、`editorTextFocus && isWorkspaceTrusted && !terminalFocus` のとき有効。

------------------------------------------------------------------------

## 引数処理

デフォルト動作:

-   引数は渡さない

------------------------------------------------------------------------

## 多言語化

-   デフォルト言語は英語（`en`）
-   VS Code の表示言語設定に応じて以下を表示する
-   日本語（`ja`）
-   中国語（`zh`）
-   フランス語（`fr`）
-   ドイツ語（`de`）
-   スペイン語（`es`）
-   上記以外の言語環境では英語にフォールバックする

------------------------------------------------------------------------

## セキュリティ挙動

ワークスペースが信頼されていない場合:

-   スクリプト実行を無効化する

------------------------------------------------------------------------

## MVP スコープ

含む:

-   JavaScript only 実装（TypeScript 不使用）
-   OS ベースの拡張子フィルタ
-   ワークスペースルートまでの親ディレクトリ走査
-   ファイル名昇順ソート
-   VS Code タスクベースの実行
-   PowerShell の実行ポリシー回避対応
-   カレントディレクトリ向けキーボード実行（1--9）
-   実行時確認なし
-   追加ログチャネルなし
-   多言語対応（`en`, `ja`, `zh`, `fr`, `de`, `es`）

含まない（将来拡張）:

-   再帰スキャン
-   カスタム拡張子設定
-   ユーザー設定項目（settings）追加
-   インタープリタ判定（bash/pwsh のプロービング）
-   実行履歴
-   引数プリセット
